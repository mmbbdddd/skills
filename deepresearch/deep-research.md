# 深度调研

**架构**：调研员（全面深入）→ 主编（聚焦简洁）→ 审计（独立验证）

---

## 执行步骤

### 1. 检测意图

收到"调研"、"深度分析"、"研究"类请求 → 激活。

### 2. 需求澄清

目的、受众、范围任一模糊 → 追问。每次 ≤3 个问题。方向不定就搜是浪费。可以多选。

### 3. 确认参数

- `agents=N`（默认 1）：研究员数量
- `purpose`：调研目的
- `audience`：受众

### 4. 创建目录

```bash
mkdir "[对象名称]"
```

### 5. 启动研究员

如果 N=1：启动 1 个研究员，使用 `dr_yjy` skill。

如果 N≥2：串行启动 N 个研究员，使用 `dr_yjy` skill。

**研究员结构**：完全相同 prompt + 完全信息隔离 + 串行启动。
三个研究员因搜索路径的自然随机差异产生不同发现——这是交叉验证的统计基础。

所有研究员同模型 → 主编交叉验证时标"同模型复现"，不视为多源独立验证，结论置信度上限"中"。

### 6. 启动主编

研究员全部完成后，启动主编，使用 `dr_zb` skill。

主编先交叉验证 → 再独立搜索 → 质量不达标打回研究员（版本号 +1，最多 3 次）。

### 7. 启动审计

主编完成后，启动审计，使用 `dr_sj` skill。

审计仅收到主报告路径，无其他上下文。不通过 → 打回主编（版本号 +1，最多 3 次）。

### 8. 汇报结果

三个阶段全部完成后，向用户汇报核心发现和审计结论。

---

## 版本号规则

每次打回重做版本号累加：v1 → v2 → v3。原版本保留。

```
[对象]_Agent1_report_v1.html   # 初始
[对象]_Agent1_report_v2.html   # 第一次打回重做
[对象]_report_v1.html          # 主编
[对象]_audit_v1.html           # 审计
```

单角色最多打回 3 次，超过 → 标记"无效"。

---

## 大型调研拆分

对象涵盖多个独立子系统时：
1. 快速扫描（30 分钟），识别子系统
2. 按子系统拆为独立子调研，各自走研究员→主编→审计
3. 汇总各子调研核心结论
